# Uncertainty Quantification — Notes (bootstrap CIs + DeLong)

## Purpose
Attach uncertainty to the headline audit metrics. **No new modeling**: this stage
resamples the already-committed out-of-fold predictions. It addresses the
reviewer concern that the pipeline reports point estimates with no confidence
intervals and no significance test for the leakage gap.

## Inputs (read-only)
- `results/pilot_gse25055/leaky_baseline_predictions.csv` (5 repeats × 306)
- `results/pilot_gse25055/nested_smo_svm_predictions.csv` (306)
- `results/external_validation_gse25065/gse25065_external_predictions.csv` (182)

## Outputs
- `tables/uncertainty/bootstrap_ci.csv` — 95% CIs for AUROC, PR-AUC, balanced
  accuracy, MCC, sensitivity, specificity (leaky / guarded / external).
- `tables/uncertainty/delta_auroc_prauc_ci.csv` — leakage gap (ΔAUROC, ΔPR-AUC)
  with paired-bootstrap CI and two-sided bootstrap p-value.
- `results/uncertainty/delong_tests.csv` — DeLong test per repeat (leaky vs
  guarded), AUROC difference.

## Method
- **Stratified percentile bootstrap**, B = 2000, global seed 20260620. Resampling
  is stratified within the pCR and RD classes so class prevalence is fixed.
- The **leaky baseline AUROC/PR-AUC point estimate is the mean across the five
  repeated-CV runs** (matching the manuscript); the bootstrap recomputes that
  mean within each resample. Operating-point metrics (balanced accuracy, MCC,
  sensitivity, specificity) pool the repeats.
- **Leakage gap** (leaky − guarded) uses a **paired** stratified bootstrap on the
  same resampled samples; the reported p-value is two-sided
  (2 × min(Pr[Δ>0], Pr[Δ<0])).
- **DeLong** test is run **per repeat** (leaky_r vs guarded nested) because the
  guarded pipeline has a single out-of-fold score per sample while the leaky
  baseline has one per repeat; this avoids collapsing the repeats into a single
  ROC that would not match the repeat-averaged point estimate.
- PR-AUC follows the project definition `PRROC::pr.curve(...)$auc.integral`
  (Davis–Goadrich integral). The R script is canonical; the Python cross-check
  uses a Davis–Goadrich implementation that matches PRROC to ~0.001–0.002.

## Reproducibility
- **Point estimates and DeLong p-values are deterministic.** AUROC and
  operating-point metrics reproduce the manuscript exactly; PR-AUC reproduces to
  ~0.001–0.002 between the Python (DG) and R (PRROC) implementations.
- **Bootstrap CIs are reproducible within each language given the fixed seed.**
  R and Python draw different resamples for the same seed value, so CI digits can
  differ slightly between `scripts/07_bootstrap_ci.R` and the Python cross-check;
  both are valid B = 2000 estimates. The committed CSVs were generated by
  `scripts/checks/07_bootstrap_ci_crosscheck.py`.

## Key findings (feed into manuscript/claim-calibration)
- **The leakage gap is positive but NOT statistically significant in this single
  cohort.** ΔAUROC = +0.0440, 95% CI [−0.0125, +0.1028], bootstrap p ≈ 0.13.
  ΔPR-AUC = +0.0362, 95% CI [−0.0483, +0.1247], bootstrap p ≈ 0.38. DeLong
  per-repeat p-values range 0.07–0.51 (median 0.18).
- Interpretation: the leaky pipeline's *direction* of inflation is consistent
  with the audit thesis, but with N = 306 and only 57 pCR cases the cohort is
  **underpowered to declare the gap significant**. The manuscript should report
  the gap with its CI and avoid implying a statistically established difference;
  the repeated-nested-CV and permutation-control analyses are needed to
  strengthen the leakage claim.
- All CIs are **wide**, driven by the small minority (pCR) class — itself an
  honest, reportable limitation.
